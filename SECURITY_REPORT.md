# 安全漏洞报告 — Flask 用户管理系统

> **项目**: flask-user-login
> **报告日期**: 2026-07-07
> **状态**: 待修复

---

## 漏洞清单

### 🔴 [高危] 漏洞 1：密码明文存储

| 项目 | 内容 |
|------|------|
| **文件** | `app.py` 第 8-23 行 |
| **描述** | `USERS` 字典中密码以明文形式存储，未做任何哈希处理 |
| **代码片段** | `"admin": { "password": "admin123", ... }` |
| **危害** | 数据库泄露、文件泄露或攻击者获取源码后，所有用户密码直接暴露 |
| **建议修复** | 使用 `werkzeug.security.generate_password_hash` 存储密码哈希 |

---

### 🔴 [高危] 漏洞 2：密码明文比对

| 项目 | 内容 |
|------|------|
| **文件** | `app.py` 第 45 行 |
| **描述** | 登录验证时直接用 `==` 比较明文密码字符串 |
| **代码片段** | `USERS[username]["password"] == password` |
| **危害** | 配合漏洞 1，攻击者可以直接用明文密码登录任意账户 |
| **建议修复** | 使用 `werkzeug.security.check_password_hash` 进行哈希比对 |

---

### 🔴 [高危] 漏洞 3：密码在前端页面明文展示

| 项目 | 内容 |
|------|------|
| **文件** | `app.py` 第 47-48 行 → `templates/index.html` 第 13-15 行 |
| **描述** | 登录后将包含密码字段的完整用户信息传入模板，首页将密码渲染在 HTML 中 |
| **代码片段** | `{{ user.password }}` |
| **危害** | 任何路过屏幕的人都能看到用户的密码；浏览器历史、页面缓存也会保存密码 |
| **建议修复** | 传递模板时过滤掉 `password` 字段；首页不展示密码 |

---

### 🟡 [中危] 漏洞 4：Secret Key 硬编码

| 项目 | 内容 |
|------|------|
| **文件** | `app.py` 第 4 行 |
| **描述** | Flask 的 `secret_key` 设置为固定的字符串 `"dev-key-2025"` |
| **代码片段** | `app.secret_key = "dev-key-2025"` |
| **危害** | 攻击者可用固定 secret key 伪造任意用户的 session cookie，无需密码即可登录 |
| **建议修复** | 从环境变量读取 `SECRET_KEY`，并提供 fallback 提示 |

---

### 🟡 [中危] 漏洞 5：缺少统一的登录认证装饰器

| 项目 | 内容 |
|------|------|
| **文件** | `app.py` 第 27-34 行 |
| **描述** | 首页仅在视图函数内部手动检查 session，没有可复用的登录装饰器 |
| **代码片段** | 无统一装饰器，每个路由需重复编写 session 检查逻辑 |
| **危害** | 后续新增路由容易遗忘登录验证，导致未授权用户访问受保护页面 |
| **建议修复** | 编写 `login_required` 装饰器统一拦截未登录请求 |

---

### 🟡 [中危] 漏洞 6：Debug 模式开启

| 项目 | 内容 |
|------|------|
| **文件** | `app.py` 第 62 行 |
| **描述** | 使用 `debug=True` 模式运行，且监听 `0.0.0.0` |
| **代码片段** | `app.run(debug=True, host="0.0.0.0", port=5000)` |
| **危害** | 页面报错时暴露完整 Python 调用栈和源码；Debugger PIN 可能被利用执行任意代码 |
| **建议修复** | 生产环境关闭 debug；通过环境变量控制是否开启 |

---

### 🟢 [低危] 漏洞 7：HTML 注释泄露默认账号

| 项目 | 内容 |
|------|------|
| **文件** | `templates/login.html` 第 1 行 |
| **描述** | 页面源码中包含 HTML 注释，写明了管理员用户名和密码 |
| **代码片段** | `<!-- 调试信息 - 默认管理员账号 用户名: admin 密码: admin123 -->` |
| **危害** | 查看网页源代码即可获得管理员密码 |
| **建议修复** | 删除该注释，或仅在开发环境通过服务器端变量控制输出 |
